Error Propagation and Uncertainty Quantification in Satellite Oceanography

A practical module on how uncertainties move through satellite retrieval chains and how to build defensible uncertainty budgets for ocean variables (SST, chlorophyll, SSH/currents, winds, and derived diagnostics).

Learning objectives

What students should be able to do
  1. Distinguish bias, random error, and correlated error—and explain why that distinction matters for satellite-derived ocean variables.
  2. Write a measurement equation (retrieval chain) for a satellite ocean product and identify dominant uncertainty terms.
  3. Propagate uncertainties through common transforms (linear combinations, ratios, logs, powers) using first-order (Taylor/Jacobian) methods.
  4. Recognize when linear propagation fails and apply Monte Carlo propagation for nonlinear algorithms and non-Gaussian errors.
  5. Handle covariance (correlated errors) and show how neglecting correlation can over- or under-estimate uncertainty.
  6. Propagate uncertainty through derivatives and averages, including effective sample size for spatial/temporal averaging and gridded products.
  7. Build an uncertainty budget for an analysis (e.g., currents from altimetry, chlorophyll anomalies, wind stress curl).

1. Why error propagation matters in satellite oceanography

Retrieval chains amplify and reshape error

Satellite oceanography is fundamentally a chain of transformations:

raw detector counts → calibrated radiance/backscatter → geophysical variable(s) → mapped/merged products → derived diagnostics (gradients, fluxes, trends)

Errors enter at every step, and what you do next can amplify or damp them:

  • Derivatives amplify noise (e.g., SSH → geostrophic velocity → vorticity).
  • Averages reduce random error only to the extent samples are independent.
  • Bias does not average away—it accumulates into trends and climatologies.
  • Correlated errors (orbit errors, striping, atmospheric correction residuals) can dominate uncertainty at large scales and low frequencies.
  • Multi-sensor fusion and assimilation require quantitative, scale-aware error models—not just quality flags.

2. Error taxonomy tailored to satellite ocean products

Random vs bias vs correlation vs sampling

2.1 Random error (noise)

Stochastic variability around the true value; often modeled as zero-mean.

  • Detector noise in radiance/brightness temperature
  • Speckle-like variability in radar backscatter (after averaging)
  • Intermittent cloud contamination behaving like outliers

2.2 Systematic error (bias)

Persistent offset (constant or slowly varying).

  • Calibration drift
  • Sea state bias mis-modeling in altimetry
  • Coastal ocean color bias due to unmodeled CDOM/sediments
  • Skin–bulk temperature differences when using SST as “bulk SST”

2.3 Correlated error (covariance structure)

Errors that are not independent across space, time, channels, or variables.

  • Space: along-track orbit error, scan-line striping, adjacency effects
  • Time: calibration drift, sampling/aliasing
  • Channels: multi-band radiance errors, cross-talk
  • Variables: coupled correction terms (e.g., SST and water vapor)
Instructor note: correlated error is the most common reason “σ/√N” fails for gridded satellite products.

2.4 Representativeness / sampling error

Mismatch between what is measured and the target quantity and scale.

  • Point buoy vs 1–25 km satellite pixel
  • Instantaneous observation vs daily mean
  • Sparse sampling (altimeter repeat) aliasing high-frequency signals

3. The measurement equation approach

Write the product as a function

3.1 Generic measurement equation

Let the derived geophysical quantity be:

$$y = f(\mathbf{x})$$

where \(\mathbf{x} = [x_1, x_2, \ldots, x_n]^T\) are inputs (radiances, brightness temperatures, correction terms, coefficients, ancillaries).

Write each input as:

$$x_i = x_{i,true} + \delta x_i$$

3.2 First-order (linearized) propagation

Using a first-order Taylor expansion about the mean:

$$\delta y \approx \sum_{i=1}^n \frac{\partial f}{\partial x_i}\,\delta x_i$$

If \(\mathbf{\Sigma_x}\) is the covariance matrix of \(\mathbf{x}\), then:

$$\sigma_y^2 \approx \mathbf{J}\,\mathbf{\Sigma_x}\,\mathbf{J}^T$$

with Jacobian \(\mathbf{J} = [\partial f/\partial x_1,\ldots,\partial f/\partial x_n]\).

Expanded scalar form:

$$\sigma_y^2 \approx \sum_i \left(\frac{\partial f}{\partial x_i}\right)^2 \sigma_{x_i}^2 + 2\sum_{i

Key point: assuming independence sets all covariance terms to zero. In satellite oceanography, that can be a big mistake.

3.3 Bias propagation

If an input has bias \(b_{x_i}=\mathbb{E}[\delta x_i]\), then (first-order):

$$b_y \approx \sum_i \frac{\partial f}{\partial x_i}\,b_{x_i}$$

Bias is often the dominant concern for trends, climatologies, and cross-sensor merges.

4. “Rules of thumb” propagation identities

Fast approximations for common transforms

These are first-order approximations; assume independence unless noted.

4.1 Linear combination

If \(y = a + \sum_i c_i x_i\), then:

$$\sigma_y^2 = \sum_i c_i^2\sigma_{x_i}^2 \quad (\text{independent})$$

4.2 Product

If \(y=x_1x_2\), then:

$$\left(\frac{\sigma_y}{y}\right)^2 \approx \left(\frac{\sigma_{x_1}}{x_1}\right)^2 + \left(\frac{\sigma_{x_2}}{x_2}\right)^2$$

4.3 Ratio

If \(y=x_1/x_2\), then:

$$\left(\frac{\sigma_y}{y}\right)^2 \approx \left(\frac{\sigma_{x_1}}{x_1}\right)^2 + \left(\frac{\sigma_{x_2}}{x_2}\right)^2$$

4.4 Power law

If \(y=x^p\), then:

$$\frac{\sigma_y}{y} \approx |p|\frac{\sigma_x}{x}$$

4.5 Log transform

If \(y=\ln x\), then:

$$\sigma_y \approx \frac{\sigma_x}{x}$$

If \(y=\log_{10}x\), then:

$$\sigma_y \approx \frac{\sigma_x}{x\ln 10}$$

5. Worked examples in satellite oceanography

From sensor space to ocean physics

Example 1: Split-window SST (thermal IR) — linear propagation

Consider a simplified split-window SST retrieval:

$$T_s = a_0 + a_1 T_{11} + a_2 (T_{11}-T_{12})$$

Rearrange:

$$T_s = a_0 + (a_1+a_2)T_{11} - a_2T_{12}$$

Assume \(a_1=1.0\), \(a_2=0.5\), \(\sigma_{11}=\sigma_{12}=0.05\,K\), independent channels.

Then:

$$\sigma_{T_s}^2 = (a_1+a_2)^2\sigma_{11}^2 + (a_2)^2\sigma_{12}^2$$

Numerically, \(\sigma_{T_s}\approx 0.079\,K\).

Interpretation: channel noise can be amplified by algorithm weights. In practice, atmospheric correction residuals, cloud leakage, emissivity/model errors, and skin–bulk differences often dominate the uncertainty budget.

Example 2: Ocean color chlorophyll from a band ratio — nonlinear propagation

Let

$$R = \log_{10}\left(\frac{R_{rs}(443)}{R_{rs}(555)}\right)$$

and a simplified relationship

$$\log_{10}(\mathrm{Chl}) = -1.5R \quad\Rightarrow\quad \mathrm{Chl} = 10^{-1.5R}$$

Assume \(R_{rs}(443)=0.010\,\mathrm{sr}^{-1}\), \(R_{rs}(555)=0.005\,\mathrm{sr}^{-1}\), and 5% uncertainties in each, independent.

For \(R=\log_{10}(x)-\log_{10}(y)\):

$$\frac{\partial R}{\partial x}=\frac{1}{x\ln 10},\quad \frac{\partial R}{\partial y}=-\frac{1}{y\ln 10}$$

Thus

$$\sigma_R^2 \approx \left(\frac{\sigma_x}{x\ln 10}\right)^2 + \left(\frac{\sigma_y}{y\ln 10}\right)^2$$

For the numbers above, \(R=\log_{10}2\approx 0.301\) and \(\sigma_R\approx 0.0307\).

Then \(L=\log_{10}(\mathrm{Chl})=-1.5R\Rightarrow \sigma_L=1.5\sigma_R\approx 0.0461\).

Finally, since \(\mathrm{Chl}=10^L\), the relative uncertainty is

$$\frac{\sigma_{\mathrm{Chl}}}{\mathrm{Chl}} \approx \ln(10)\,\sigma_L \approx 0.106 \;(10.6\%)$$

Interpretation: small reflectance uncertainties can become larger relative uncertainties in chlorophyll after ratios and logs. Band errors are often correlated via atmospheric correction, which changes the result.

Example 3: Altimetry SSH → geostrophic velocity — derivatives and correlated errors

For geostrophic flow (NH):

$$u = -\frac{g}{f}\frac{\partial \eta}{\partial y}$$

Finite difference:

$$\frac{\partial \eta}{\partial y} \approx \frac{\eta_2-\eta_1}{\Delta y} \quad\Rightarrow\quad u \approx -\frac{g}{f}\frac{\eta_2-\eta_1}{\Delta y}$$

Let \(\sigma_{\eta_1}=\sigma_{\eta_2}=\sigma\). If correlation between \(\eta_1\) and \(\eta_2\) is \(\rho\), then:

$$\mathrm{Var}(\eta_2-\eta_1)=2\sigma^2(1-\rho)$$

Key lesson: for gradients/differences, positively correlated errors can cancel. Ignoring covariance can make uncertainty estimates badly wrong (sometimes too large, sometimes too small).

Example 4: Scatterometer wind speed → wind stress

Bulk formula:

$$\tau = \rho_{air} C_D U^2$$

Sensitivity:

$$\frac{\partial \tau}{\partial U} = 2\rho_{air} C_D U$$

So \(\sigma_\tau \approx (\partial \tau/\partial U)\,\sigma_U\).

Rule check: because \(\tau\propto U^2\), a 5% wind error implies about a 10% stress error (first-order).

6. When first-order propagation is not enough

Monte Carlo as a robust default

First-order (Jacobian) propagation assumes small errors and local linearity. It can fail for:

  • strongly nonlinear retrievals (e.g., SSS, complex atmospheric correction)
  • thresholding/branching logic (cloud masks, quality filters)
  • heavy-tailed errors/outliers (cloud edges, sun glint leakage)
  • values near detection limits (log-space artifacts)

6.1 Monte Carlo propagation

Algorithm:

  1. Specify distributions for inputs \(x_i\) (include covariance if relevant).
  2. Draw \(M\) samples \(\mathbf{x}^{(k)}\).
  3. Compute \(y^{(k)}=f(\mathbf{x}^{(k)})\).
  4. Estimate mean, variance, and percentiles from \(\{y^{(k)}\}\).

Correlated inputs: if \(\mathbf{x}\sim\mathcal{N}(\mu,\Sigma)\), sample via Cholesky:

$$\Sigma = LL^T,\quad \mathbf{x}=\mu+L\mathbf{z},\;\mathbf{z}\sim\mathcal{N}(0,I)$$

7. Propagation through averaging, binning, and gridding

Why σ/√N often lies

7.1 Mean of independent samples

If \(\bar x = \frac{1}{N}\sum_{i=1}^N x_i\) with independent errors:

$$\sigma_{\bar x} = \frac{\sigma_x}{\sqrt N}$$

7.2 Effective sample size for correlated samples

With correlation, uncertainty reduction is weaker. One common approximation:

$$N_{eff} \approx \frac{N}{1 + 2\sum_{k=1}^{N-1}\left(1-\frac{k}{N}\right)\rho_k}$$

Then:

$$\sigma_{\bar x} \approx \frac{\sigma_x}{\sqrt{N_{eff}}}$$

7.3 Weighted averages and composites

If \(y=\sum_{i=1}^N w_i x_i\) with \(\sum_i w_i=1\):

$$\sigma_y^2 = \mathbf{w}^T\Sigma\mathbf{w}$$

For independent errors: \(\sigma_y^2=\sum_i w_i^2\sigma_i^2\).

8. Multi-sensor fusion and objective mapping (advanced)

Minimum-variance linear ideas

8.1 Combining two independent estimates

If \(x_1, x_2\) are unbiased with variances \(\sigma_1^2, \sigma_2^2\), the minimum-variance linear estimator is:

$$x = \frac{x_1/\sigma_1^2 + x_2/\sigma_2^2}{1/\sigma_1^2 + 1/\sigma_2^2}$$

with variance

$$\sigma_x^2 = \left(\frac{1}{\sigma_1^2}+\frac{1}{\sigma_2^2}\right)^{-1}$$

8.2 Objective mapping viewpoint

Objective analysis methods explicitly model signal covariance and observation-error covariance. Many gridded products provide a mapping error variance field. Interpret it as:

“Expected analysis error under the assumed covariance model,” not a universal guarantee of true error.

9. A practical workflow for satellite uncertainty budgets

How to be defensible
  1. Define the target quantity and scale. (Instantaneous vs daily mean; 1 km vs 1° boxes; etc.)
  2. Write the measurement equation. Make every correction explicit.
  3. Inventory error sources (random + bias + correlation scale) in a budget table.
  4. Choose a propagation method: Jacobian for smooth steps; Monte Carlo for nonlinear/thresholded steps.
  5. Carry covariance when differencing, gridding, or fitting trends.
  6. Validate with matchups—carefully. Matchup RMSE includes representativeness and collocation error.
  7. Communicate uncertainty honestly (what’s included, what’s not, and assumptions).

10. Common pitfalls

What goes wrong most often
  1. Mixing variance and standard deviation (forgetting to square/√).
  2. Assuming “quality flag = uncertainty.” Flags indicate conditions, not quantitative σ.
  3. Ignoring covariance when taking differences/gradients (currents!) or averaging.
  4. Treating matchup RMSE as satellite uncertainty without representativeness analysis.
  5. Using \(\sigma/\sqrt N\) with \(N\)=pixels even though pixels are correlated.
  6. Propagating random error but forgetting bias (especially for trends).
  7. Unit mistakes in derivatives (meters vs kilometers; degrees latitude; etc.).

11. Exercises (with brief solution sketches)

Homework / exam-ready
Problem 1 (core): Linear propagation

An SST retrieval is \(T_s = a_0 + 1.2T_{11} + 0.3(T_{11}-T_{12})\). Given \(\sigma_{11}=0.04\,K\) and \(\sigma_{12}=0.05\,K\) (independent), compute \(\sigma_{T_s}\).

Sketch: rewrite as \(T_s=a_0+(1.2+0.3)T_{11}-0.3T_{12}\), then sum coefficient-squared times variances.

Problem 2 (core): Log-ratio propagation

Let \(R=\log_{10}(x/y)\) with \(x=0.012\pm 4\%\), \(y=0.006\pm 6\%\). Compute \(\sigma_R\). Then if \(\log_{10}(\mathrm{Chl})=b_0+b_1R\) with \(b_1=-1.2\), compute relative \(\sigma_{\mathrm{Chl}}/\mathrm{Chl}\).

Sketch: use \(\sigma_{\log_{10}x}=\sigma_x/(x\ln 10)\), sum in quadrature, multiply by \(|b_1|\), convert from log to linear via \(\ln 10\).

Problem 3 (core/advanced): Correlated SSH errors

You estimate geostrophic velocity from two SSH points separated by \(\Delta y\). Assume \(\sigma_{\eta_1}=\sigma_{\eta_2}=\sigma\) and correlation \(\rho\). Derive \(\sigma_{\Delta\eta}\) and show how \(\sigma_u\) depends on \(\rho\).

Sketch: \(\mathrm{Var}(\eta_2-\eta_1)=2\sigma^2(1-\rho)\); then \(\sigma_u=(g/f)\sigma_{\Delta\eta}/\Delta y\).

Problem 4 (advanced): Averaging with correlation

You average \(N=30\) daily SST anomalies with lag-1 autocorrelation \(\rho_1=0.7\) (assume AR(1)). Approximate \(N_{eff}\approx N\frac{1-\rho_1}{1+\rho_1}\). Compare \(\sigma_{\bar x}\) using \(N\) vs \(N_{eff}\).

Sketch: \(N_{eff}\approx 30\cdot(0.3/1.7)\approx 5.29\); uncertainty reduction is far weaker than \(1/\sqrt{30}\).

Problem 5 (lab-style): Monte Carlo propagation

Implement Monte Carlo uncertainty for \(\mathrm{Chl}=10^{-1.5\log_{10}(R_{rs}(443)/R_{rs}(555))}\) given distributions for Rrs uncertainties (optionally include correlated band errors). Compare Monte Carlo \(\sigma\) to Jacobian \(\sigma\).

Sketch: sample Rrs, compute Chl ensemble, report mean/σ and percentile bounds; test sensitivity to correlation.

Appendix: Minimal Python templates (for a lab)

Optional starter code

A) Finite-difference Jacobian (scalar output)

import numpy as np

def jacobian_fd(f, x, eps=1e-6):
    x = np.array(x, dtype=float)
    y0 = f(x)
    J = np.zeros_like(x)
    for i in range(len(x)):
        dx = np.zeros_like(x)
        dx[i] = eps * max(1.0, abs(x[i]))
        J[i] = (f(x + dx) - y0) / dx[i]
    return J

# Example usage:
# y = f([T11, T12])  -> SST

B) Monte Carlo propagation with optional covariance

import numpy as np

def monte_carlo(f, mu, Sigma, M=100000, seed=0):
    rng = np.random.default_rng(seed)
    L = np.linalg.cholesky(Sigma)
    z = rng.standard_normal((M, len(mu)))
    x = mu + z @ L.T
    y = np.array([f(xi) for xi in x])
    return {
        "mean": y.mean(),
        "std": y.std(ddof=1),
        "p16": np.percentile(y, 16),
        "p84": np.percentile(y, 84),
    }
Implementation note: For performance, vectorize f if possible. For complex retrieval chains, Monte Carlo is often the most robust way to carry nonlinearities and non-Gaussian errors.
